# Conversation Summarizer

## Purpose
Creates structured summaries of sessions for future context loading.

## When to Summarize
**Automatic:** End of significant session, before long break, when context switching.

**On Request:** User says "Summarize what we did", "Create session summary".

## Summary Structure
```markdown
# Session Summary: YYYY-MM-DD
**Duration:** HH:MM
**Project:** [project-name]

## Goals
[What user wanted to accomplish]

## Accomplished
- [Done 1]
- [Done 2]

## In Progress
- [Started but not finished]

## Key Decisions
- [Decision 1]
- [Decision 2]

## Key Findings
- [Discovery 1]

## Issues Resolved
- [Problem] → [Solution]

## Follow-ups
- [ ] [Action item 1]
- [ ] [Action item 2]

## Files Modified
- [[file]] - [what changed]

## Learned
[New knowledge gained]

---
*Generated by Conversation Summarizer*
```

## Quick Summary (shorter)
```markdown
## Session: YYYY-MM-DD
**Project:** [name]
**Done:** [1 sentence summary]
**Next:** [what to do next]
```

## Storage
```
~/.claude/memory/summaries/
├── YYYY-MM-DD_[project]_[hash].md
└── [project]/[year]/[month]/[day]_[topic].md
```

## How to Use Past Summaries
```bash
ls -t ~/.claude/memory/summaries/[project] | head -3
```

## Output
When summarizing: "Session saved. Summary: [1 sentence overview]"

## Constraints
- Summaries should be scannable
- Include action items with checkboxes
- Keep under 500 words for quick loading
- Include file references for later
